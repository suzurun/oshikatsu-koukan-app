# 1. サービス概要・ビジョン

## 1.1 ミッション

> 「推し愛でつながり、AIが守る、信頼の市場」

推し活（コンサートグッズ・アイドルグッズ等）を起点とした、**AIによる価格透明化と詐欺防止を核心差別化とするC2Cマーケットプレイス**を構築する。

## 1.2 解決する問題

| 問題 | 現状 | 本サービスの解決策 |
|---|---|---|
| ぼったくり価格 | 相場を知らずに高値掴み | AI相場分析で全プラットフォーム横断価格表示 |
| 発送詐欺 | 代金を払ったのに商品が届かない | エスクロー決済で発送確認まで代金保留 |
| 偽物・コピー品 | 偽物を正規品として販売 | AI真贋判定 + 画像ハッシュ照合 |
| 現地交換のドタキャン | 約束した相手が現れない | デポジット制度 + 信用スコア連動ペナルティ |
| 相場の急変 | 公演前後の価格を把握できない | イベントドリブン価格予測アラート |

## 1.3 フェーズ展開

| フェーズ | 内容 | ターゲット | 期間 |
|---|---|---|---|
| **Phase 1** | 推し活特化（物々交換＋売買） | コンサート・アイドルグッズ | 〜12ヶ月 |
| **Phase 2** | エンタメ全般拡大 | ゲーム・アニメ・コレクターズ | 12〜24ヶ月 |
| **Phase 3** | 全品目C2C | メルカリ相当・全品目 | 24ヶ月〜 |

## 1.4 スケール目標

| 指標 | Phase 1 | Phase 3 |
|---|---|---|
| 登録ユーザー | 100,000人 | 10,000,000人 |
| 月間取引件数 | 10,000件 | 5,000,000件 |
| 出品数 | 50,000件 | 1,000,000,000件 |

# 4. AIシステム要件

## 4.1 AI価格分析パイプライン

### 処理フロー

```
① 画像受信（Supabase Edge Function: analyze-price）
        ↓
② Google Vision API
   → 商品カテゴリ判定
   → 商品名・シリーズ・キャラクター同定
   → 状態スコア（0〜1）算出
        ↓
③ Ximilar API
   → コレクターズアイテム専門価格データ取得
   → 類似商品の過去成約価格
        ↓
④ 自社DB照合
   → market_price_data テーブル（メルカリ・ヤフオク・eBay）
   → 自社成約履歴（transactions + listings）
        ↓
⑤ Claude API（claude-sonnet-4-6）
   → 需給スコア算出
   → 希少性スコア算出
   → 3段階価格提案生成
   → 需要予測（30日）
        ↓
⑥ ai_price_analyses テーブルに保存 → クライアントに返却
```

### 価格提案仕様

| 提案種別 | 算出方法 | 表示 |
|---|---|---|
| 早期売却価格 | 市場中央値 × 0.85 | 「すぐ売れる」帯 |
| 推奨価格 | 市場中央値 | 「推奨」帯 |
| プレミア価格 | 希少性スコア×係数 | 希少度51以上のみ表示 |

### 希少性スコア算出

```
希少性スコア = (需要数 / 供給数) × 状態係数

需要数: ウィッシュリスト登録数 + 検索ヒット数（30日）
供給数: 全プラットフォームの同一商品出品数
状態係数: new=1.0 / like_new=0.9 / good=0.7 / fair=0.5

スコア分類:
  0〜20:  common（通常）
  21〜50: uncommon（やや希少）
  51〜80: rare（希少）
  81〜100: super_rare（超希少）
```

### パフォーマンス要件

| 項目 | 目標値 |
|---|---|
| AI分析レスポンス | 3秒以内（P95） |
| 画像認識精度 | 商品カテゴリ正答率90%以上 |
| 価格提案誤差 | 市場中央値との乖離±30%以内 |

## 4.2 詐欺防止AIシステム

### 多層防御アーキテクチャ

| レイヤー | 機能 | 実装方法 |
|---|---|---|
| Layer 1 | eKYC（本人確認） | TRUSTDOCK API |
| Layer 2 | 行動分析（リアルタイム異常検知） | fraud_behavior_logs + ルールエンジン |
| Layer 3 | 画像AI（偽物・盗用検知） | Ximilar + 画像ハッシュ照合 |
| Layer 4 | エスクロー決済（発送詐欺防止） | Stripe Connect |
| Layer 5 | ソーシャルグラフ分析 | IP・デバイスID・行動クラスタリング |

### 詐欺シグナルと閾値

| シグナル | 検知方法 | アクション |
|---|---|---|
| バースト出品（1h以内10件以上） | 統計的逸脱 | fraud_risk_scores 更新 + 要確認フラグ |
| 異常ログイン（普段と異なる地域） | 地理的異常 | 本人確認SMS送信 |
| 重複画像（別出品から使い回し） | pHash照合 | 出品ブロック + 通知 |
| 外部誘導文言 | Claude NLP | メッセージ送信ブロック |
| 詐欺ネットワーク（同一IP/デバイス群） | グラフ分析 | 関連アカウント一括調査 |

### 詐欺リスクスコア対応

| スコア | 対応 |
|---|---|
| 0〜30 (low) | 通常通り取引可 |
| 31〜60 (medium) | 追加確認メッセージ表示 |
| 61〜80 (high) | 取引前に追加確認（SMS/顔認証） |
| 81〜100 (critical) | 取引停止 + 管理者通知 |

## 4.3 AI自動モデレーション

| 対象 | 機能 | 精度目標 |
|---|---|---|
| 出品画像 | 禁止商品・性的コンテンツ検知 | False Positive 1%以下 |
| 商品説明文 | 詐欺ワード・禁止ワード検知 | Claude NLP |
| 異常価格 | 相場の5倍超を自動フラグ | ルールベース |

## 5.1 取引ステータス遷移

```
[申請] requested
   ↓ 出品者承認
[合意済み] agreed
   ↓ KYC未完了の場合
[KYC待ち] kyc_required → agreed（KYC完了後）
   ↓ Stripe Escrow決済
[デポジット保留中] deposit_held
   ↓ 双方が「発送準備OK」
[発送解禁] shipping_unlocked
   ↓ 出品者が発送
[出品者発送済み] shipped_by_seller
   ↓ 購入者が受取確認
[受取確認済み] receipt_confirmed
   ↓ エスクロー解放
[完了] completed

※分岐:
  任意タイミング → cancelled（双方合意キャンセル）
  任意タイミング → auto_cancelled（72h未発送）
  shipped_by_seller → disputed（紛争申立）
  disputed → dispute_resolved（運営裁定）

交換取引:
  shipping_unlocked → shipped_by_seller（出品者発送）
                   → shipped_by_buyer（購入者発送）
                   → both_shipped（双方発送）
                   → delivered → completed
```

## 5.2 取引種別別フロー

| 取引種別 | 特記事項 |
|---|---|
| sale（売買） | 購入者のみ発送確認。エスクロー決済必須。 |
| exchange（交換） | 双方が発送。デポジット双方。双方受取確認で完了。 |
| local_exchange（現地交換） | 当日モードと連携。デポジット500円。15分不在で申請可。 |

## 5.3 エスクロー設計

```
購入者が決済申請
  ↓
Stripe Payment Intent 作成（amount: agreed_price）
  ↓
escrow_records に status='holding' で記録
  ↓
出品者が発送 → 購入者が受取確認
  ↓
Stripe キャプチャ実行（out品者のStripe Accountへ送金）
  ↓
escrow_records.status = 'released'
transactions.status = 'completed'

紛争発生時:
  escrow_records.status = 'holding' のまま凍結
  運営が裁定 → 'released' or 'refunded'

デポジット没収（未発送違反）:
  escrow_records.status = 'forfeited'
  信用スコア -15%
  相手方へ返金
```

## 5.4 発送期限管理

| イベント | タイミング | アクション |
|---|---|---|
| 発送解禁 | shipping_unlocked 時 | shipping_deadline = NOW() + 72h を設定 |
| 第1リマインド | deadline - 48h | 通知: shipping_reminder |
| 第2リマインド | deadline - 24h | 通知: shipping_reminder（緊急） |
| 自動キャンセル | deadline 超過 | auto_cancelled + 未発送側ペナルティ |

# 10. 法務・コンプライアンス要件

| 法律 | 対応方針 | 実装 |
|---|---|---|
| **古物営業法** | 3万円超の取引でKYC必須。プラットフォームは「場の提供者」として古物商非該当。 | kyc_required ステータスで自動判定 |
| **特定商取引法** | 継続的販売者（月10件以上等）への事業者表示案内 | is_verified_seller フラグ + 表示義務案内 |
| **資金決済法** | エスクロー運営のための第三者型前払式支払手段の登録 | Stripe Connect経由で法的対応 |
| **犯罪収益移転防止法** | 疑わしい取引の報告義務。記録7年保存。 | 詐欺リスクスコア高いユーザーの取引監視 |
| **個人情報保護法** | KYC書類はKMS暗号化。復号は承認制。ログ5年保存。 | Supabase KMS + RLS |
| **著作権法** | 著作権侵害品（同人・コピー品）の出品禁止を規約明記 | AI画像審査 + 通報システム |

### 禁止事項（規約に明記）
- 売買差額請求禁止
- 外部連絡先強要禁止
- 著作権侵害品の出品禁止
- アーティスト公式との誤認表示禁止
- 18歳未満の子の直接取引禁止（親管理下のみ）
