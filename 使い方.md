# 完全自動復旧システム - 使い方ガイド

## 🎯 このシステムでできること

レート制限エラーが発生したAPI呼び出しを、**完全に復旧するまで自動的にリトライ**します。

### 何が自動化されるか？

1. **レート制限エラーの検出** - HTTP 429エラーを自動検出
2. **自動リトライ** - エラーが復旧するまで自動的に再試行
3. **待機時間の自動計算** - APIが指定する待機時間を自動的に尊重
4. **復旧確認** - 2回連続成功で復旧を確認
5. **ログ記録** - すべての試行を記録
6. **統計情報** - 成功回数、エラー回数などを自動収集

async function example() {
  const stats = new StatsCollector();
  const monitor = new Monitor({ stats, updateInterval: 2000 });

  try {
    monitor.start();

    const data = await fetchJSONWithAutoRecoveryEnhanced(
      'https://api.example.com/data',
      {},
      { stats, monitor }
    );

    monitor.stop();
  }
}

## 🔧 設定

### 設定ファイル (`auto-recovery.config.json`)

```json
{
  "autoRecovery": {
    "retryForever": true,      // 完全復旧まで無限リトライ
    "baseDelay": 1000,         // ベース待機時間（1秒）
    "maxDelay": 300000         // 最大待機時間（5分）
  },
  "logging": {
    "enabled": true,           // ログを有効化
    "saveToFile": true         // ログをファイルに保存
  },
  "notifications": {
    "enabled": true,           // 通知を有効化
    "onRecovery": true         // 復旧時に通知
  }
}
```

## 📝 実際の使用シナリオ

### シナリオ1: APIがレート制限に達した

**問題**: APIを呼び出したら「Rate limit reached」エラーが発生

**解決**: このシステムを使うと、自動的に復旧まで待機してリトライ

```javascript
// 通常のコード（エラーが発生する）
const response = await fetch('https://api.example.com/data');
// ❌ Rate limit reached エラー

// このシステムを使う（自動復旧）
const data = await fetchJSONWithAutoRecovery('https://api.example.com/data');
// ✅ 自動的に復旧まで待機して成功
```

### シナリオ2: 複数のAPIを順次呼び出し

**問題**: 複数のAPIを呼び出す必要があるが、レート制限が心配

**解決**: 各API呼び出しが自動的に復旧まで待機

```javascript
const endpoints = [
  'https://api.example.com/users',
  'https://api.example.com/posts',
  'https://api.example.com/comments',
];

for (const endpoint of endpoints) {
  // 各呼び出しが自動的に復旧まで待機
  const data = await fetchJSONWithAutoRecovery(endpoint);
  console.log('データ取得:', data);
}
```

### シナリオ3: 長時間実行されるバッチ処理

**問題**: 大量のAPI呼び出しが必要で、レート制限が発生する可能性がある

**解決**: 統計とモニタリングで進行状況を確認しながら自動復旧

```javascript
const stats = new StatsCollector();
const monitor = new Monitor({ stats });

monitor.start();

// 大量のAPI呼び出し
for (const url of urls) {
  await fetchJSONWithAutoRecoveryEnhanced(url, {}, { stats, monitor });
}

monitor.stop();
stats.displayStats(); // 統計情報を確認
```

## 💡 よくある質問

### Q: どのくらい待機しますか？

A: 完全に復旧するまで自動的に待機します。APIが指定する待機時間（Retry-Afterヘッダー）を自動的に尊重します。

### Q: 手動操作は必要ですか？

A: いいえ、完全に自動化されています。一度実行すれば、復旧まで自動的に待機します。

### Q: ログはどこに保存されますか？

A: デフォルトで `recovery-log.json` に保存されます。設定で変更可能です。

### Q: 統計情報はどうやって確認しますか？

A: `stats.displayStats()` で表示するか、`stats.saveToFile()` でファイルに保存できます。
